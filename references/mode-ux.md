# Mode UX

This product has two user-facing modes.

## Default mode

- active unless the user explicitly enables advanced mode
- safest supported path
- prefer search, static extraction, and supported site extractors
- should be the stated mode in normal usage

## Advanced mode

- must be explicitly enabled by the user
- applies only to the current local skill runtime state
- still limited to public content and limited dynamic pages
- does not widen the product promise to login-required or challenge-heavy sites

## Expected user messaging

When advanced mode is enabled, say that clearly.

When advanced mode is unavailable or unnecessary, say that clearly.

When a task is outside product scope, explain the boundary directly instead of hinting that one more retry might solve it.
